# AI 报"没做"必须升级为"没做,因为 X,需要 Y"

## 规则

**任何 AI(飞书 AI / 我 / 未来的第三方 AI)**,当你要报「没做 / 部分做 / 假成功」时,**同一句话里必须给足三段**:

1. **卡点** — 具体到工具 / 数据 / 权限 / 加载量 / 冲突,不能只说"我没做"
2. **解法候选** — 1-3 个,你自己判断可行的
3. **要天蓝做什么** — 明确动作请求(批准 A / 提供 X / 决定不做)

**"我没做"是问题描述,不是问题解决**。只报问题不给解法 = 把包袱扔回天蓝,让她自己去摸原因。

## Why

**2026-07-08 教训**:飞书 AI 累积 53 次「老实说没做」(铁律 #5),但每次只报事实不报原因。天蓝直到 7-8 才自己摸出根源 = 工作空间垃圾文件太多导致加载超时干扰判断。如果飞书在第 1 次「老实说」时就把「卡在垃圾文件多」+「需要清理权限」一起报,天蓝 3 天前就能给权限,不会累积到 53 次。

**打断链的隐形代价**:天蓝要自己反复猜"为啥这次又没做"= 隐形打断,消耗她的判断力。

## How to apply · 标准模板

```
我说了做 X
实际:没做 / 部分做 / 假成功
卡点:__________(具体到工具/数据/权限/加载量)
解法候选:A __ / B __ / C __
需要天蓝:__________(批准 A / 提供 Y / 决定不做)
```

## HARD RULES

- **不给卡点** = 你在把问题扔回天蓝,自己没动脑
- **不给解法** = 你只报警不救火
- **不给动作请求** = 天蓝要猜你要什么,又是打断链
- **卡点必须具体**:"我加载慢"不算,"工作空间 606K + 12 个 daily artifacts 过 7 天导致 fetch 20s+"才算
