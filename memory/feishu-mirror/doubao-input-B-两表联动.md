## 我的痛点 · 两表现在还是割裂的

### 痛点 1 · 想看某配饰"全流程"要翻 4 张表
- 主表看总状态
- TL 表看模型进度
- 挂点表看挂点进度
- 特殊任务表看 DB 进度
- 待办表看今天要做什么

### 痛点 2 · workload 判断难
- 某人这周忙不忙 · 只能数待办条数
- TL 表活跃任务人员字段 80% 填了 · 20% 缺
- 外包人员没挂 · workload 视图不完整

### 痛点 3 · v1.1 跑测暴露的联动问题(实测 1 轮)

**已发现 8 个具体问题**:
1. **负责人从文本提取误填率高** · T081 内容"郑一凡已改完提交" · 实际制作人是王智超
2. **关键词匹配跨任务错误关联** · 仅"耳机"就把 T065 关联到 P143EA_HD
3. **内容字段与负责人不同步** · 改了负责人 · 内容还写旧人名
4. **历史记录父任务 ID 断档** · T034/T027 等历史没有父任务 ID
5. **父任务 ID 命名不统一** · T079 = "ST-生日" · 不是标准编号
6. **截图文字误识别** · AI 从截图文件名提取人名/编号
7. **TL 表人员字段影响 workload** · 20% 未填(外包)
8. **主表 DDL vs 待办计划截止** · 含义不同 · 天蓝混淆
   - 主表 DDL = 配饰最终交付日(如 P143EA_HD DDL = 7/31)
   - 待办计划截止 = 当前动作最晚完成日(如 T081 = 7/18)

### 痛点 4 · TL 表本身也有问题(9 个)

**核心 3 个**:
1. **阶段过粗** · "制作中"同时含白模/高模/拓扑/UV · 无"当前动作"字段
2. **版本号缺失** · P143EA_HD 已到 v02 · 但 TL 表没"当前版本"字段
3. **审核信息没有** · 不知道当前审核人 · 反馈发出多久了 · 催过几次
