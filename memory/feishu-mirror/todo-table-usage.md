| 值 | 含义 |

## 7 视图分工

### 1 · 分流区(评判优先级)
**入口** · 筛选:新待办 = ✅
- 飞书 AI 甩进来 · 天蓝审内容 · 勾优先级 · 取消「新待办」✅
- 审完自动跳出

### 2 · 工作待办
筛选:🔴急急 + 安排中 · 按类别分组
你今日/明日要打的仗

### 3 · 追踪
筛选:🟡追踪 · 按类别分组
别人在做 · 你只盯进度

### 4 · AI 协作
筛选:类别 = AI 协作
GitHub Claude 端待办独立入口 · 分组:安排中 / 已完成

### 5 · 沉淀区
筛选:「备注和沉淀」有内容
- 完成/归档后 · 天蓝写结论/经验
- GitHub Claude 也可写沉淀建议
- **双人共享字段**

### 6 · 已归档
筛选:状态 = 归档
历史留档 · 不再照会

### 7 · 后台
无筛选 · 全部数据 · 搜索用

## 关键工作流

### A · 甩待办(飞书 AI SKILL)
1. 天蓝一句话 → 飞书 AI 自动开单
2. 自动填:ID / 内容 / 附件 / 超链接 / 状态=安排中 / 来源=天蓝 / 创建日 / 「新待办」= ✅
3. 类别关键词匹配 · 不准就空
4. 优先级留空(等天蓝勾)
5. 甩进「分流区」等审

### B · 分流(天蓝)
1. 打开分流区
2. 审内容对不对
3. 勾优先级(4 选 1)
4. 取消「新待办」✅
5. 记录自动跳到对应视图(工作待办/追踪)

### C · 急+追踪流转(核心逻辑)
一件事分 2 段:
- **第 1 段** · 你要推动 → 优先级=🔴急急 · 状态=安排中
- **第 2 段** · 推完等对方 → 优先级=🟡追踪 · 状态=待回复
- 对方回后 → 优先级=🔴急急(收尾) · 状态=安排中
- 收尾完 → 状态=已完成

### D · 沉淀(天蓝 + GitHub Claude 双向)
- 完成/归档后 · 天蓝在「备注和沉淀」字段写结论
- GitHub Claude 在对话里主动提议"这条建议沉淀"
- 天蓝手动落飞书「备注和沉淀」字段
- 每周三 19:30 · aily schedule 下行到 memory/feishu-mirror/todo-table-mirror.md
- GitHub Claude 未来会话读到

### E · 归档(天蓝手动)
- 完成后长期不会再照会 · 状态=归档
- 归档项无优先级
- AI 无自主归档权 · 天蓝拍板

## 铁律

**所有 AI 建的东西 · 最后一步必是天蓝复核 · 复核后才算可靠信息**
