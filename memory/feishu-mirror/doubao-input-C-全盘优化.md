### 主题 2 · 配饰表 A.5 优化建议(5 项 · 待评估)

#### 2.1 建立主表-子表双向同步

**现状** · 主表 → TL 单向镜像(每小时) · TL 字段更新不回主表

**建议** · 主表字段更新 → 推送到 TL 对应记录 · 保持任务名/版本/编号一致

**问豆包**:
- 双向同步会不会导致数据循环冲突?
- 如果做 · 谁优先(主表 vs TL)?
- 有没有更简单的方案(如只做 A → B 单向 · 但更频繁)

#### 2.2 状态机自动流转

**想法** · 基于群消息关键词自动更新主表状态
- "模型上传了" → 主表状态推进
- "挂点做完了" → 主表状态推进
- "引擎合入了" → 主表状态推进

**问豆包**:
- 群消息关键词识别准确率有保障吗?(现在已有原画/模型截图自动录入 · 参考经验)
- 错识别的兜底机制?(如天蓝定期人工校验)
- 状态是否应该允许回退?(如"重做")

#### 2.3 挂点表与主表联动

**现状** · 挂点表孤立 · 状态变化不回主表

**建议** · 挂点状态变更 → 自动回写主表"挂点状态"字段

**问豆包**:
- 主表加"挂点状态"字段 · 还是完全依赖挂点表(不回写主表)?
- 视图上怎么让"一页纸看完"最方便?

#### 2.4 统一版本号规范

**问题** · 群消息版本号格式多样 · 1230 / 2610 / V03
**建议** · 统一 obt-261230 或 V03 格式 · 减少 AI 解析模糊

**问豆包**:
- 强推统一格式 · 团队接受度?
- 或者 AI 端做多格式兼容(现在的做法)· 但增加误判风险?
- 哪个方向更可持续?

#### 2.5 特殊任务到期提醒

**建议** · 特殊任务流程加 DDL 监控 · 逾期自动提醒

**问豆包**:
- 提醒频率?(逾期 1 天 · 3 天 · 每天)
