**日期**：用户 2026-07-06 交付。

**核心定义**：所有"可选碎片事件"在玩家未主动触发、错过时间窗口或操作异常时，系统**自动执行默认行为**，保证主线流程完整可通、不卡关、不报错，**玩家无感知内容缺失**。

**通关标准（两项都过才算合格）**：
1. **强约束** —— 全程无报错、无卡关、无空白内容、无逻辑矛盾
2. **软约束** —— 玩家全程只点收银台也能获得完整、自洽、无割裂感的体验，感知不到"有内容被跳过了"

## ⚠️ 架构决策一致性提示

以下用例基于**严格线性 CASHIER_TIMELINE**（严格单客流：任意时刻店内仅存在 1 位待完成流程的客人，前一位客人完整离店后，才会触发下一位客人进店）编写。

- **C01 用例：「护士帮老人拿牛奶」交叉彩蛋 —— v1.0 暂不落地，v2+ 版本预留**（老人完整离店后护士才进店，不存在同店场景）
- **C02 用例：客流串行推进，任意时刻仅一位客人在店，不存在同时结账冲突**

**v1.0 版本范围明确**：不做多客同店、不做交叉彩蛋、不做多线并行。这些延后迭代，跟当前架构解耦，不出现在机制/文案/测试任何环节。

# 七、必须默认初始化的 flag 列表（V01 依据）

代码里 `state.world` 或 `cashierArc` 初始化时**必须**显式赋这些默认值，不能等 beat effects 才 set：

```js
state.world = {
  catBowl:      'ignored',   // 门口猫碗
  milkHelp:     'none',      // 帮老人拿奶
  odenState:    'off',       // 关东煮
  seatHeater:   'off',       // 司机窗边
  doorGap:      'none',      // 少年门缝
  cashService:  'normal',    // 老人结账（有默认档所以给 normal）
  nurseCheckout:'normal',    // 护士结账默认档
  driverCheckout:'normal',   // 司机结账默认档
  chargerState: 'none',      // 少年手机
  cleanerCheckout: null,     // 未触发就是 null（进阶段 3 前）
  driverAltCheckout: null,   // 未触发就是 null（进阶段 5 前）
  nurseObserved: false,      // 观察类默认 false
  teenObserved: false
};
```

**别 lazy init**——V01 用例就是查这个。lazy 会出 `undefined`。
