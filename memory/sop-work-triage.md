# 任务分级 · 减少无效打断

**目标**：分清必接 / 可延 / 可婉拒，减少烦躁。**核心**：不是"该不该接"，是"接的方式"。

---

## 3 类必接（今天真实场景）

### A · 教学式 ⚠️ 高成本

**特征**：技术连环问 / 需要 handhold
**例子**：陈明星 UV/材质/染色 连环问 (30-60 min)
**判断**：外派/新人 + 第一句是"XX 怎么做"

**应对（3 步）**：
1. **立刻回** "半小时后回你，先看下 memory 里的 [sop-颜色/命名/文件提交] 有没有答案"
2. **答完顺手沉淀**：把答案的核心逻辑写成 SOP（不用完美，先扔到 memory）
3. **下次同问题**："去看 sop-XXX，看完还有问题再来"

**目标**：3-5 次之后 陈明星式教学时间降 70%（她自己能查）

---

### B · 咨询式 ✅ 低成本

**特征**：要个快速决定
**例子**：麦大麦 "章凯晖空了用嘛" (2-5 min)
**判断**：**老板 / 组长 / PM** + 第一句是"XX 情况 / XX 好不好"

**应对**：
- **立即回**（tier 1 必回）
- 不确定就回："我 15 min 后拉一下情况回你"，别硬答

---

### C · 提醒式 ✅ 低成本

**特征**：heads-up 你漏了什么 / 别错了
**例子**：山吹 "RY 头发别漏 / 面具登记表" (1-3 min)
**判断**：内部同事 + 第一句是"XX 做了吗 / 别忘 XX"

**应对**：
- **秒回"收到"**（回复本身 30 秒）
- 具体动作**可延后到当前任务段结束**（比如现在做完这段代码 / 校完这个色，再去处理提醒）
- 加进 today's list，别忘

---

### D · 转发/流转式 ⚠️ 中成本（正在自我消除中）

**特征**：你在做 middleman（文件 / 消息 / 需求 中转）
**例子**：外包发文件给你 → 你下载 → 转发外派
**判断**：来源方**不能直接**跟目标方对接 → 你被迫居中

**应对（3 步）**：
1. **立即转发**（当下别拖，越拖你后面越忙）
2. **同时给外派 educate**："下次这类文件你直接去 `<地方>` 看，别等我转"
3. **累计 3-5 次同样的转发** → 说明制度问题，推动**直接对接**（申请权限 / 建共享盘）

**长期目标**：**消除自己作 middleman**（你已经在做，好）。见 [feedback-outsource-file-mess]。

**HARD RULE**：转发是低价值劳动。**永远不要把它当"我的正事"**——它是流程 bug 的补丁。

## 2 类可推（虽然今天没例子，但常见）

### D · 求助式（跨组 / 非你岗位）

**特征**：别组临时找帮忙
**判断**：来自**非你日常协作圈** + 事情不属于配饰审核 / 挂点管理

**应对**：
- ✅ "这个我不熟，找 X 更快" —— 委婉转
- ✅ "半小时后回你" —— 如果确实要接
- ❌ 别硬接（陷入无关的坑）

**HARD RULE**：如果是**至尊菜刀 / 老板**发的，无论"是否你岗位"都是**必接**（tier 1）。

### E · 闲聊 / 打招呼

**特征**：跟工作无关
**应对**：
- 当日下班前**批量回**
- 别每来一条切换

## 判断树（3 秒定位）

```
消息进来 → 看谁 + 看第一句
├─ 老板/菜刀/麦大麦/牛二 → tier 1 立即回（不用往下判断）
├─ 陈明星/王智超技术问题 → 教学式 A
│   └─ 沉淀触发！ 重复问题该写 SOP 了
├─ 组长要决定 → 咨询式 B
├─ 内部同事 "XX 别漏" → 提醒式 C
├─ 别组临时求助 → 判断是不是你岗位 → D
└─ 无关消息 → E 批量处理
```

## 回复模板（复制即用）

**教学式 · 引导查文档**：
> 「先看下 sop-XX 那个文档，看完还有具体问题再来找我」

**咨询式 · 拖延回答**：
> 「我 15 min 后拉下情况回你」

**提醒式 · 秒收 + 后置动作**：
> 「收到，做完这段就处理」

**求助式 · 委婉转**：
> 「这个我不熟，你找 X 更快，他做过类似的」

**求助式 · 接但拖延**：
> 「半小时后回你，现在手头有个东西没完」

**闲聊 · 延迟批量**：
> （下班前统一回）

---

## 沉淀触发规则（防止再被教学式吃时间）

**每周五复盘**问自己 3 个问题：
1. 这周教学式打断有几次？（陈明星 / 王智超 / 外派问技术问题的合计次数）
2. 有没有**同一个问题被问 ≥ 2 次**？→ 立刻写成 SOP
3. 已有 SOP 但外派没查？→ 下次问的时候先甩链接

**教学式的 SOP 命名规范**：`sop-<人>-<主题>.md` 或 `sop-<主题>.md`（如 `sop-uv-materials.md`）

---

## HARD RULES · 不许打破

- **乘凉 / 菜刀** 任何时候任何渠道 = **立即回**（哪怕你在 flow 中）
- **麦大麦** = 5 分钟内回（除非你在开会）
- **邮件** = 2 小时内回，不做实时
- **私人微信** = 下班前统一

---

## 每天早上 2 分钟

看今天日程 → 划出**能保护的 2-3 小时深度工作段** → 在飞书状态改"专注中"→ 只 tier 1 的人能打断你

- ✅ 拒绝 = "我在专注一个卡了很久的东西，一小时后回"（不用负疚）
- 极其重要 = 找搭档兜底（feedback-anxiety-affects-judgment）

## 元规则 · 天蓝对 AI 也别形成打断链

**2026-07-07 教训**：飞书 AI 承诺 30 min 完成 2a，天蓝在这 30 min 里连发 4 条新任务（applink 短链 / P001CH / record link 等），飞书被拽下来，1h42min 后 0 完成。

**规律**：跟人相同 —— **连续新消息 = 打断链 = 承诺无法兑现**。

**How to apply**：
- 对飞书 / 我 拍完任务后 → **给它一个不被打断的时段**（哪怕只有 10 min）
- 中间冒出新想法 → **先记在自己本子上**（或跟另一个 AI 说），等这个任务出结果再统一发
- 我这边 / 飞书 应该学会说 "排队进 X 类，先做完手头 Y" 而不是真去接新任务

**双向责任**：AI 自己要守铁律不假成功，用户也要给 AI 保护时段。**任何一方失守链条就断**。
