# 5 大块 + 1 每周 · 每日检视 SOP

**设计原则**：**责任块 = 每天需要检视的稳定维度**。每天早上 10 min 走一遍这 5 块，产出今日待办。**不是逐条都要做，是识别今天在哪块要重点花时间**。

---

## 每日 5 块

### 🔵 1. 配饰审核 + 反馈 + 人力调度

**流程一体的 3 件事**（都跟"这批活谁做什么"绑在一起）。

**每日 checklist**：
- [ ] 昨晚 / 今早外包外派提交的东西：**看了吗**？
- [ ] 需要**给反馈**的还没回的有几个？
- [ ] 当前时间节点：手上活来不来得及？有**DDL 风险**要提前预警吗？
- [ ] **人力盘点**：
  - 谁在原计划？
  - 谁**空了**（原本任务做完 / 别的没排上）→ 补什么活？
  - 谁**空不出来**（预期能做但实际卡住了）→ 谁替他做急事？
- [ ] **急的东西**给谁做？

**心态提醒**：**"谁空了"是每天都变的**，不是安排一次就完事。承认这是**碎片但必须**的工作，不要期望能一次性搞定。

---

### 🟠 2. 挂点 & BS 跟进

**每日 checklist**：
- [ ] **挂点疑难杂症群** Pin 有没有新的？
- [ ] Pin 里的今天能处理几个？
- [ ] **默认发型**有没有待确认的？（BS 优先算这个，见 [sop-attachment-point-workflow]）
- [ ] BS **批量算**的时机到了吗？（默认发型确认之后才批量）

**长期优化想法**（open item）：**群机器人**自动追 Pin 状态、提醒待办 —— 未来展开设计

---

### 🟡 3. 跨组对接答复（原画 / 绑定 / 别组）

**这块最零散，最容易漏**。

**每日 checklist**：
- [ ] **原画（菜刀等）review** 有没有等我回的？
- [ ] 原画说的"要改" → **改了吗**？谁改？什么时候改完？（**必须跟进到闭环**）
- [ ] **绑定组**回来的问题 → 我这边处理了吗？
- [ ] 有没有**别组等我答复**的？（想到就登记，不然会忘）

**回应模板**：
- 没时间处理 → "收到，X 时间前给你答复"（先给期望）
- 已处理 → "已改，看下 XX 文件"（+ 具体位置）

---

### 🟢 4. V9 数据表维护

**这个表是很多任务的统计、统筹 → 必须及时**。

**每日 checklist**：
- [ ] 昨天做完的东西 → **状态更新**了吗？
- [ ] 原画交的**新东西** → 主表加了吗？
- [ ] **挂点表 / TL 表**镜像同步了吗？（1:1 镜像，走工作流不用手动，见 [project-accessory-pipeline]）
- [ ] status 有没有明显**不对**的（比如卡了一周没动的）？

---

### 🟣 5. 手串项目（额外）

**每日 checklist**：
- [ ] 手串今天推到什么进度？
- [ ] 有没有卡点需要处理？

（这项细节还没沉淀，等有更多信息可以升级成独立 project 文件）

## ⏰ 每周（不是每天）· 沉淀外派文档

**为啥不放每日**：**"来不及写"** 本身就是每日 checklist 塞不进去的证据。要**专门排时段**：
- 周五下班前 1-2 小时
- 或者周末找一段完整时间
- 需要**关消息 / 关群通知**（否则又被打断）

**每周 checklist**（周五复盘时走一遍）：
- [ ] 这周被反复问几次同样问题？→ **写成 SOP**（教学式 → 沉淀触发，见 [sop-work-triage]）
- [ ] 已开的坑（比如**链条珠子外包制作文档**、[sop-piece-check-list-needed]）今天推进多少？
- [ ] 旧 SOP 有没有需要更新的？

---

## 每天早上 10 分钟怎么走

1. **打开这份 SOP** → 5 块顺次扫一遍（每块 1-2 min）
2. **每块产出** 0-3 个今日待办
3. **打包**成 today's list（可以贴在飞书状态 / 桌面便签 / 纸上）
4. **保护深度工作段**（见 [sop-communication-priority] tier 分级 + [sop-work-triage]）

## 关键：不要每天想"我该做什么"

**"我该做什么"是这个 SOP 应该回答的**。你的脑力应该花在**判断**（怎么调度 / 怎么反馈 / 怎么优先），而不是**发现**（今天有啥要做）。**发现 = 每天早上 10 min 走这个清单**。
