**规则**：任何写给天蓝或天蓝转给外派的文档，都要**轻量级操作流程卡片**风格。

**Why:** 天蓝要转给一线外派同事对着执行的文档，理论长文没人看。外派需要"步骤 1、步骤 2"式的清单，快速定位关键动作。

**How to apply:**
- **步骤编号清晰**（1./2./3. 而不是散段落）
- **❌ / ✅ 速查表** 展示禁忌和推荐做法
- **核心禁忌加粗突出**（一眼扫到）
- 少用长段落，多用要点
- 例子用真实项目编号（P163FA、YG098 之类），不用抽象术语
- **反面例子**：不要写"根据美学原理，颜色的对比度对整体质感有重要影响..."
- **正面例子**：`✅ 黑要够黑（比 #202020 深） / 亮要够亮（高光 > #F0F0F0） / ❌ 不要整体发灰`

## 2026-07-08 补 · 扁平化 + 分工

- **能一行别多行,能扁平别嵌套**:模板/规则优先一句话说完,不搞"规则→Why→How→HARD RULES→相关"五段套
- **正面**:「没做 X · 卡在 __ · 需要你 __」(一行)
- **反面**:同一条规则展开成 5 段 40+ 行(2026-07-08 我第一版 [[feedback-ai-blocker-disclosure]] 就踩过,49 行 → 22 行 → 用户还嫌多)

**分工原则**:天蓝负责发散(想到啥说啥,发散是她的思维方式,别压),我负责收束(整合零散点、精简、扁平化输出)。她给素材,我给成品;她给方向,我给一行。**能不新建文件就扩现有的**,MEMORY.md 越薄越好。

## 2026-07-08 补 · 待答清单顶部挂

**规则**: 天蓝单次回复通常只答一部分, 剩下的问题不会主动重提。AI 每次回复必须在最顶部挂「待你回的清单」, 列所有还没被答的问题, 直到答完。

**Why**: 天蓝之前遇到多个未答问题被 AI 忘掉。挂清单 = AI 记忆负担, 不是天蓝的。

**How to apply**:
- 每次回复最上面: `## 待你回的清单` + bullet 列所有待答项
- 天蓝答一条, 下次回复从清单划掉
- 清单空了不显示
- 新问题 append 进去

**反面**: 追问"另外那 3 个你怎么想" —— 逼天蓝翻上下文
**正面**: 顶部列「洞 2 / 洞 3 / 洞 4 · 还没答」 —— 她扫一眼就知
