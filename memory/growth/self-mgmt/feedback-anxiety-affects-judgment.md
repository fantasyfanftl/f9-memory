## 现象

**天蓝自己很急的时候容易看不好反馈**。焦虑状态下判断力会下降。

需要**别人帮忙**：
- 帮忙找关键文件
- 帮忙审一遍

**Why:** 急 → 注意力窄 → 容易只盯当前那件事、看漏别的 / 判断力不稳。这是普遍规律不是 personal weakness。承认它才能设计流程。

## How to apply

**给自己设兜底**：

1. **发现自己在急** → **不做重大判断**（急的时候尤其不要给外派下最终反馈，容易改错方向再返工）

2. **常备"救火搭档"** — 平时约好一两个同事："我告诉你我急了，你帮我看一眼这件事"，互相 backup

3. **一次只处理一件事** — 急的时候把别的浏览器 tab 关掉、群消息通知关，物理上强制单线程

4. **急的项目单独做 checklist** — 越急越不能凭感觉走，写在纸上照着做能防遗漏

5. **事后复盘** — 事情过去了记录一下"这次急是啥导致的"，多次积累后能识别早期信号
