用户在 lycheeGame 项目里做的 `merge-town.html` 是仿制「求合体」(4399 老游戏,忘川 2014-04-02 发布)。**注意区分它和「合并小镇」/「合成大西瓜」是完全不同的玩法**,首次做的时候被带偏过一次。

**正确规则(用户已确认):**
1. **三消合成**:3+ 个相同物品「横 / 竖 / 折(L 形)」相连(4-连通,不含斜)自动合并到下一级
2. **商店购买放置**:玩家从商店买最低几级物品,点击空格子放置
3. **通关条件**:把整张地图的可放置空地全部铺满 —— 不是达成某个高等级
4. **连锁**:合成产生的新物品若又凑齐 3 个,继续连锁
5. 高级物品(大树以上)只能靠合成得来,商店买不到
6. 地图带障碍物(石头/湖),形状因村庄而异
7. 有金币经济:合成奖励金币,金币用于买物品

**Why:** 用户最初描述「求合体」时我按合并小镇(2-drag merge)实现,被明确纠正。三消 + 铺满地图是这类老游戏的核心创新,不能搞混。

**How to apply:**
- 如果以后再提「求合体」/ 类似 4399 老游戏,先按这套规则实现
- 迁移到别的合成类游戏时先问清:2-drag 还是 3-adjacency?物理下落还是网格?通关目标是什么?
- 参见 [[lychee-game-site]] 了解部署路径,`merge-town.html` 已上线
