## 一、核心 4 条强制重绘规则（永久生效）

### 1. 统一视点规则
全场景固定单一原点：店员 1.5m 人眼高度、50mm 标准焦距，仅水平 Y 轴环视旋转，垂直视角无俯仰；5 张场景互相保留 15° 画面重合区域用于锚点衔接。

### 2. 跨场景锚点规则
内置统一参照物（承重立柱、通铺地脚线、吊顶灯盘），相邻画面重合区域锚点像素级对齐，作为空间连续判断唯一依据。

### 3. 光影色彩统一规则
固定 3000K 店内主暖光、冷柜 8000K 局部冷光、门外低亮度 6000K 环境光；全图统一色温 3200K、饱和度 ≤ 20%，同材质 RGB 波动差值 ≤ 5%。

### 4. 四项验收校验强制规则
重绘资产必须依次通过：空间透视校验 → 光影色彩校验 → 锚点拼接校验 → 滑动交互校验，四项全部合格方可替换上线。

## 二、3 处底层冲突说明（不可忽视前置约束）

### 1. Hotspots 坐标冲突
v1.0 架构约定沿用 legacy 热区坐标；完整重绘场景后 18 个交互热区必须全部重标定，同时完整重跑 [[midnight-store-selfcheck]] Checklist B 顶视图对照校验。

### 2. AI 生图成品约束
沉淀反馈 [[feedback-ai-gen-instability]]：AI 生成画面透视、光影稳定性不足，仅可作为灵感草稿，禁止直接作为交付成品；优先 Blender 3D 建模方案。

### 3. 版本开发并行红线
v1.0 主线验收优先级最高，**禁止双线并行开发**场景重绘；场景重绘完整计划归档入 v1.1 backlog，启动前置条件为 v1.0 稳定版全量验收发布。

## 三、标准化落地执行路线（推荐时序）

1. **当前 v1.0 周期**：完全沿用 legacy 旧场景图，完成 17 项功能验收 + 11 条 Legacy QA，稳定发布；**不启动任何场景重绘工作**。
2. **v1.0 上线后，v1.1 前置筹备（2-4h）**：Figma / Blender 绘制 1:1 店内平面布局蓝本，锁定所有空间坐标、摄像机旋转角度，仅输出线稿基准，不渲染成品图。
3. **美术资产生产（优选 Blender 手工建模，工时 5-10 天）**：基于统一蓝本渲染全套 5 张连贯场景，补充叙事所需特写视角素材；次选 AI 线稿 + PS 统一光影作为临时过渡资产。
4. **资产替换全量校验**：替换新场景图 → 重标定全部 Hotspots 坐标 → 四轮视觉校验 → 更新适配 CSS 过渡动效。
5. **v1.1 迭代验收**：新增视觉连贯性专项验收项，校验通过后发布新版本。
