## 问题类型

历史配饰在流程规范化之前留下的坑，现在做染色 / 迭代时反复被咬。

## 具体案例

### 1. 原色制作时没有 ID 导致图案对不上
染色时**变成大坑**，最著名的例子：**薯片头套**。

### 2. 外派改了 UV 导致之前的工作白做
UV 一改，之前基于旧 UV 做的所有贴图 / 效果都得重来。

## 影响范围

多少个配饰有这类遗留问题？**目前不清楚**（open item）。有些平时不动就没事，一动就爆。

## How to apply

**短期**：
- 遇到一个记一个，遇到的 fix 步骤沉淀成 note，避免下次重头研究
- 外派**不要随便改 UV**（如需改，必须先通知天蓝评估影响）

**中期**：
- 建**历史资产健康档**：列出已知有问题的配饰 + 问题类型 + 是否已 fix
- 高频用到的历史资产 → 主动 fix，别等触发

**长期**：
- 新做的东西**从建模阶段就规范**（ID / 命名 / UV）
- 这是 project-accessory-pipeline 里"三表体系"要解决的一部分
