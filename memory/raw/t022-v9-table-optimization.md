## 已知痛点(2026-07-09 12:30 天蓝口述)

### 痛点 1 · Formula 特殊情况不好改

**现状**: V9 表多个状态字段用 formula 自动算

**问题**:
- 遇到特殊情况(状态不该按公式跑的时候)· 改不动
- 要 debug"哪些条件导致公式不触发预期结果"
- 追溯麻烦

### 痛点 2 · Single Select 联动断

**如果**改成 single select(手填):
- ✅ 特殊情况直接改
- ❌ 表和表之间的状态无法联动
- 手动同步累 · 会漂移

## 权衡矛盾

**根源**: V9 表承担 2 个矛盾职责
- **生产工具**: 要灵活应对异常 → single_select 更好
- **数据源**: 要精确联动 → formula 更好

## 备选方案(周六讨论用)

### 模式 A · 双字段(推荐初步方案)
- Formula 计算状态 = 系统状态(联动用)
- 手动覆盖状态 = 特殊情况填,空则以 formula 为准
- 最终状态 = if 手动有值 → 用手动 · else → 用 formula

### 模式 B · 全 single select + 联动脚本
- 全手填 · 用飞书自动化在表间联动
- 灵活但脚本要维护

### 模式 C · 主表 formula + 特殊处理视图
- 主表 formula
- 建"特殊处理"视图 · 加覆盖字段
- 有值就 override · 类似 A 但更轻
